# 策略規格文件（可擴充多策略）

## 目的
本文件記錄目前策略規則，作為後續調整、回測與多策略組合依據。

## 適用範圍
- 資料來源：SQLite DB（由 Shioaji 更新）
- 週期：依使用者選擇（1m/5m/15m/30m/60m/1d）
- 時段：日盤 / 夜盤 / 全盤
- 僅用於回測與訊號視覺化，不保證實盤績效

## 重要限制
- 策略只適用「當日結果」。
- 若訊號跨日才成立（例如昨天收盤與今天開盤才形成進場條件），該訊號不計入。

## 共通限制
- 策略只適用「當日結果」。
- 若訊號跨日才成立（例如昨天收盤與今天開盤才形成進場條件），該訊號不計入。
- 每個交易時段收盤前 30 分鐘：
  - 不再產生新的進場訊號
  - 若仍有持倉，於觸及「距收盤 30 分鐘」的第一根 K 棒強制平倉
- 即時模式不做「最後一根強制平倉」。
- 回測模式會在最後一根收盤強制平倉（僅用於回測統計）。

# 策略1：MA60/MA100 撐壓進場

## 架構說明
- 每個策略以獨立函式實作，彼此解耦。
- UI 可複選策略，後續可擴充為多策略組合判斷進出場。
- 目前已上線策略：`MA60/MA100 撐壓進場`。

## 名詞定義
- 關鍵K（第 N 根）：以 buffer=10 判斷觸及，
  - `Low - 10 <= MA60 <= High + 10` 或
  - `Low - 10 <= MA100 <= High + 10`
- 第 N+1 根：用來判斷吞噬與進場的 K 棒
- 多方吞噬：第 N+1 根 Close > 第 N 根 max(Open, Close)
- 空方吞噬：第 N+1 根 Close < 第 N 根 min(Open, Close)

## 方向判斷（和 MA60 有關）
- 多方：MA60_slope > 0 且當前 Close > MA60
- 空方：MA60_slope < 0 且當前 Close < MA60

## 進場規則
### 多單
同時滿足以下條件才進場：
1. 關鍵K 觸及 MA60 或 MA100（含 buffer）
2. 第 N+1 根 Close > 第 N 根 max(Open, Close)
3. 關鍵K 的 Open 與 Close 都 > 其觸碰到的 MA（MA60 或 MA100）
4. MA60_slope > 0，且第 N+1 根 Close > MA60
5. 不跨日（第 N 與第 N+1 根必須在同一交易日）

### 空單
同時滿足以下條件才進場：
1. 關鍵K 觸及 MA60 或 MA100（含 buffer）
2. 第 N+1 根 Close < 第 N 根 min(Open, Close)
3. 關鍵K 的 Open 與 Close 都 < 其觸碰到的 MA（MA60 或 MA100）
4. MA60_slope < 0，且第 N+1 根 Close < MA60
5. 不跨日（第 N 與第 N+1 根必須在同一交易日）

## 出場規則
- 多單：當第 N+1 根 Close < 第 N 根 min(Open, Close) 時出場
- 空單：當第 N+1 根 Close > 第 N 根 max(Open, Close) 時出場

## 風控規則
- 套用「共通限制」。
- 新增「自動風控參數」：
  - 取最近 365 天（至少 1 年）回測交易，統計每筆 `max_loss_points` / `max_profit_points`
  - 以分位數推導：
    - 最大虧損限制（停損）：`max_loss_points` 的 80 分位
    - 獲利啟動門檻（啟動動態停利）：`max_profit_points` 的 65 分位
    - 動態停利回撤距離：`最大虧損限制 * 0.5`
  - 若樣本不足（max_loss 或 max_profit 任一少於 12 筆），則不啟用自動風控，回到原始 N/N+1 出場規則

## 動態停利/停損執行細節
- 進場後若浮動虧損達到「最大虧損限制」，立即停損出場
- 進場後若浮動獲利先達到「獲利啟動門檻」，啟用動態停利
- 啟用後以「歷史最高浮盈 - 當前浮盈」計算回撤，回撤達到門檻即停利出場
